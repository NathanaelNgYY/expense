# M3 + M5 + M6 UI Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the tab bar honest navigation semantics with keyboard support (M3), make the desktop presentation of the 430px column intentional (M5), and fix observed short-screen defects with a permanent regression guard (M6).

**Architecture:** M3 is a self-contained change to `TabBar.tsx` (attribute swap + one keyboard handler on the `<nav>`). M5 is CSS-only inside one new `@media (min-width: 700px)` block in `index.css`, driven entirely by existing theme tokens. M6 is a two-phase task: an audit script produces a defect report, then only reported defects get CSS fixes; a new e2e check at 320×568 locks the result in.

**Tech Stack:** React 19 + TypeScript, Vitest + React Testing Library (fireEvent, NOT userEvent), Playwright (mobile-chromium project), plain CSS with custom properties.

**Spec:** `docs/superpowers/specs/2026-07-17-m3-m5-m6-design.md` — read it first.

## Global Constraints

- Work on branch `feat/m3-m5-m6-ui-polish`; never commit to `main` (protected).
- Repo test convention: colocated `*.test.tsx` run by Vitest; RTL with `fireEvent` — do not add `@testing-library/user-event`.
- CSS bundle budget is **13 KiB gzip** (current actual 12.3). If `npm run build && node scripts/check-bundle-size.mjs` fails, STOP and escalate — never raise the budget.
- Mobile (<700px) must render identically: M5 CSS lives only inside `@media (min-width: 700px)`.
- M6 fixes only observed defects from the audit report — no speculative compact CSS.
- Inactive tabs get NO `aria-current` attribute (not `aria-current="false"`).
- Arrow keys move focus only; they never call `onChange`.

```tsx
// src/components/TabBar.tsx
import type { KeyboardEvent } from 'react'
import { ChartColumn, CirclePlus, House, Lightbulb, Settings } from 'lucide-react'

export type Tab = 'home' | 'history' | 'add' | 'insights' | 'settings'

interface Props {
  active: Tab
  onChange: (tab: Tab) => void
}

// Arrow/Home/End move focus between the five destinations (wrapping) as a
// progressive enhancement over the natural tab order; activation stays on
// Enter/Space/click. Focus movement must never trigger navigation.
function handleKeyDown(event: KeyboardEvent<HTMLElement>) {
  const { key } = event
  if (key !== 'ArrowLeft' && key !== 'ArrowRight' && key !== 'Home' && key !== 'End') return

  const buttons = Array.from(event.currentTarget.querySelectorAll<HTMLButtonElement>('button'))
  const index = buttons.indexOf(document.activeElement as HTMLButtonElement)
  if (index === -1) return

  event.preventDefault()
  const next =
    key === 'ArrowRight' ? (index + 1) % buttons.length
    : key === 'ArrowLeft' ? (index - 1 + buttons.length) % buttons.length
    : key === 'Home' ? 0
    : buttons.length - 1
  buttons[next].focus()
}

export default function TabBar({ active, onChange }: Props) {
  return (
    <nav className="tab-bar" aria-label="Main navigation" onKeyDown={handleKeyDown}>
      <button
        type="button"
        aria-label="Home"
        aria-current={active === 'home' ? 'page' : undefined}
        className={active === 'home' ? 'active' : ''}
        onClick={() => onChange('home')}
      >
        <House className="tab-icon" aria-hidden="true" strokeWidth={2.3} />
        <span>Home</span>
      </button>
      <button
        type="button"
        aria-label="History"
        aria-current={active === 'history' ? 'page' : undefined}
        className={active === 'history' ? 'active' : ''}
        onClick={() => onChange('history')}
      >
        <ChartColumn className="tab-icon" aria-hidden="true" strokeWidth={2.3} />
        <span>History</span>
      </button>
      <button
        type="button"
        aria-label="Add entry"
        aria-current={active === 'add' ? 'page' : undefined}
        className={`tab-add${active === 'add' ? ' active' : ''}`}
        onClick={() => onChange('add')}
      >
        <CirclePlus className="tab-icon" aria-hidden="true" strokeWidth={2.3} />
        <span>Add</span>
      </button>
      <button
        type="button"
        aria-label="Insights"
        aria-current={active === 'insights' ? 'page' : undefined}
        className={active === 'insights' ? 'active' : ''}
        onClick={() => onChange('insights')}
      >
        <Lightbulb className="tab-icon" aria-hidden="true" strokeWidth={2.3} />
        <span>Insights</span>
      </button>
      <button
        type="button"
        aria-label="Settings"
        aria-current={active === 'settings' ? 'page' : undefined}
        className={active === 'settings' ? 'active' : ''}
        onClick={() => onChange('settings')}
      >
        <Settings className="tab-icon" aria-hidden="true" strokeWidth={2.3} />
        <span>Settings</span>
      </button>
    </nav>
  )
}
```

```js
// Short-viewport audit (M6): screenshots + overflow/clipping metrics for every
// primary screen at SE-class sizes across all four themes.
import { chromium } from 'playwright'
import { mkdirSync } from 'node:fs'

const BASE = 'http://127.0.0.1:4173'
const OUT = '.superpowers/sdd/short-audit'
const THEMES = ['original-dark', 'deep-sea', 'copper-current', 'berry-circuit']
const VIEWPORTS = [{ width: 320, height: 568 }, { width: 375, height: 667 }]
const today = new Date().toLocaleDateString('en-CA')

// Navigation clicks per screen, from the Home tab. Primary control per screen
// is what a user must be able to reach and tap.
const SCREENS = [
  { name: 'home', nav: [], primary: { role: 'button', name: 'Add entry' } },
  { name: 'add', nav: [['button', 'Add entry'], ['button', '5']], primary: { role: 'button', name: /Save|Add for/ } },
  { name: 'history', nav: [['button', 'History']], primary: { role: 'button', name: 'Next month' } },
  // Insights has no single action control; its primary is the page heading, which
  // only needs to be visible — minHeight 0 exempts it from the 44px control rule.
  { name: 'insights', nav: [['button', 'Insights']], primary: { role: 'heading', name: /Insights/, minHeight: 0 } },
  { name: 'settings', nav: [['button', 'Settings']], primary: { role: 'button', name: "Reset This Month's Data" } },
  { name: 'settings-automatic', nav: [['button', 'Settings'], ['button', /Automatic Tracking/]], primary: { role: 'button', name: 'Back' } },
  { name: 'settings-budget', nav: [['button', 'Settings'], ['button', /Budget & Categories/]], primary: { role: 'button', name: /Save|Back/ } },
  { name: 'settings-appearance', nav: [['button', 'Settings'], ['button', /Appearance/]], primary: { role: 'button', name: /Back/ } },
  { name: 'settings-data', nav: [['button', 'Settings'], ['button', /Data & Backup/]], primary: { role: 'button', name: /Back/ } },
]

mkdirSync(OUT, { recursive: true })
const browser = await chromium.launch()
const findings = []

for (const theme of THEMES) {
  for (const viewport of VIEWPORTS) {
    const page = await browser.newPage({ viewport })
    await page.route('**://*.supabase.co/**', route => route.abort())
    await page.addInitScript(([t, d]) => {
      localStorage.clear()
      localStorage.setItem('budget_entries', JSON.stringify([
        { id: 'audit-1', amount: 12, category: 'lunch', note: 'Lunch', date: d },
      ]))
      localStorage.setItem('budget_config', JSON.stringify({
        monthlyIncome: 1200, lunch: 264, transport: 50, savings: 400,
        investments: 250, others: 236, buffer: 236,
      }))
      localStorage.setItem('budget-tracker-theme-v2', t)
    }, [theme, today])

    for (const screen of SCREENS) {
      await page.goto(BASE + '/')
      try {
        for (const [role, name] of screen.nav) {
          await page.getByRole(role, { name, exact: typeof name === 'string' && name.length <= 2 }).first().click()
        }
        await page.waitForTimeout(400)
        const slug = `${screen.name}-${theme}-${viewport.width}x${viewport.height}`
        await page.screenshot({ path: `${OUT}/${slug}.png`, fullPage: false })

        const overflowX = await page.evaluate(() => document.documentElement.scrollWidth > window.innerWidth)
        const primary = page.getByRole(screen.primary.role, { name: screen.primary.name }).first()
        await primary.scrollIntoViewIfNeeded().catch(() => {})
        const box = await primary.boundingBox()
        const minHeight = screen.primary.minHeight ?? 44
        const clipped = !box || box.height < minHeight || box.y + box.height > viewport.height + 1
        if (overflowX || clipped) {
          findings.push({ screen: screen.name, theme, viewport: `${viewport.width}x${viewport.height}`,
            overflowX, primaryBox: box, symptom: overflowX ? 'horizontal overflow' : 'primary control clipped or <44px' })
        }
      } catch (error) {
        findings.push({ screen: screen.name, theme, viewport: `${viewport.width}x${viewport.height}`,
          symptom: `navigation failed: ${String(error).slice(0, 120)}` })
      }
    }
    await page.close()
  }
}

await browser.close()
console.log(JSON.stringify(findings, null, 2))
console.log(`\n${findings.length} finding(s). Screenshots in ${OUT}/`)
```

Note on `primary.boundingBox()` heights: `boundingBox` reflects post-scroll position; a control is "clipped" only if, after `scrollIntoViewIfNeeded`, it still does not fit above the viewport bottom (the +1 tolerates subpixel rounding).
